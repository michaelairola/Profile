What the hell is a server?

If you are reading about servers, I'm going to assume you're old enough to not be offended by swear words. Most people know very little about servers, yet they are the foundational building blocks to the Internet as we know it. If you have plenty of knowledge and understanding of what servers are, good for you. Continue to the next section if you want to learn how to build your own! For the rest of us, I'm going to start with the basics: What the hell is a server?

A server is a computer set up to interact with other computers by sending and recieving data. When the computer sends data to another computer, it is said to be "serving" the data (hence the name). When the computer is recieving data from another computer, we call the message that is recieved a "request".

Let's look at the classic browsing-the-web scenario; You know, the thing you were probably doing about 5 minutes ago? When you click a link, your computer sends a request to the server associated with the url of that link. From here, the server will send data to your computer, typically in the form of an html file, for your browser to read and interpret into the website you see! Pretty neat stuff right?

Now before you say "Wow that's so simple! Why does Wikipedia make it sound so complicated?" I have to tell you that this is a very basic (yet common) example of what a server does. Servers can also send and recieve e-mail (known as e-mail servers), send and recieve files (file servers) and do many other things as long as the proper server-software is installed on the computer. The example above focuses on Hyper Text Transfer Protocol (HTTP) servers, because hypertext is the type of data that browsers read and convert into websites! So there you have it, the basics of what a server is!
